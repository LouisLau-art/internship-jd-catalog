## Codebase Understanding

### Architecture Overview

The project maintains a clean separation between:
1.  **Raw Data**: JSON/CSV exports in `data/` generated by scripts in `scripts/`.
2.  **Logic**: `scrape_campus_jobs.py` serves as the central orchestrator for merging data from various sources (urllib APIs, Playwright, or Firecrawl results).
3.  **Docs**: Structured company notes and shortlists in `docs/`.
4.  **Resumes**: Highly specific Python scripts in `resumes/sources/` that render tailored resumes based on JD requirements.

### Key Patterns Discovered

- **`bunx` Preference**: The user strongly prefers `bunx` over `npx` for CLI tools.
- **Dynamic Content Handling**: When direct API reverse-engineering fails (e.g., XHS or Baidu), fallback to `agent-browser` for DOM inspection or Firecrawl for smart scraping.
- **Tencent API Persistence**: Tencent's `searchPosition` API is stable but sensitive to headers (User-Agent, Referer).
