# 牛客刷题追踪功能设计

## 概述

在现有 `internship-jd-catalog` 项目中增加牛客网刷题追踪功能。核心原则是 **保持简单，聚焦学习**，不做复杂的自动化爬取，只做本地手动记录整理，帮助用户系统化整理刷题过程中的知识点和错题。

## 背景与动机

- 用户正在准备华为等大厂AI岗校招笔试，需要在牛客网刷题
- 当前零散的记录不利于复习和进度追踪
- 需要一个轻量级的组织方式，不占用太多精力（避免"为了做工具而做工具"的拖延）
- 项目已有求职招聘信息整理和简历生成能力，增加刷题整理符合项目定位

## 设计决策

### 1. 范围边界

**包含:**
- 刷题笔记文档，记录题目、选项、解答、知识点
- 代码解答文件存放（Python/C++等）
- 简单的进度统计（已刷题量，正确率统计可选）

**不包含:**
- 牛客网自动爬虫/API抓取 - 需要登录认证，容易失效，分散精力
- 复杂的Web UI查看界面 - Markdown原生可读性足够

### 2. 存储结构

```
project-root/
├── docs/
│   ├── nowcoder_practice.md          # 主刷题笔记文档（按时间顺序记录）
│   └── superpowers/specs/
│       └── 2026-03-29-nowcoder-practice-tracker-design.md  # 本设计文档
├── {题目解答文件直接放根目录}
│   ├── kmeans_clustering.py          # 华为笔试 - 用户分群（已放这里）
│   ├── ridge_regression.py           # 华为笔试 - 缺失值填补（已放这里）
│   └── ...                           # 未来继续放这里
```

**理由:**
- 与项目已有做法保持一致（算法题解答已经放在根目录）
- 单笔记文档便于搜索和复习
- 不需要复杂的多层目录，增加维护负担

### 3. 笔记格式规范

`docs/nowcoder_practice.md` 采用以下结构：

```markdown
# 牛客刷题记录

## 刷题进度

- 总题数：XX
- 已完成：XX
- 正确率：XX%

---

## 题目列表

### 1. [题目标题](链接) - [分类] - 完成日期

**题目描述：**
```cpp
// 代码题目粘贴这里
```

**选项：**
- A. ...
- B. ...
- C. ...
- D. ...

**我的答案：**
**正确答案：**

**知识点总结：**
- ...
- ...

---

### 2. 下一道题...
```

## 组件接口

不需要复杂接口，只有:
1. 手动编辑 `docs/nowcoder_practice.md` 添加新题目
2. 在根目录添加代码解答文件（如果是编程题）
3. 可选：运行简单Python脚本统计刷题进度

## 错误处理

- 无外部依赖，所以基本没错误需要处理
- 用户手动录入，错了直接编辑修正即可

## 测试策略

- 单元测试针对进度统计脚本（如果实现）
- 笔记格式只需要人类可读，不需要测试

## 待以后扩展

如果以后需要，可以扩展:
- 按分类分文件
- 自动生成HTML刷题看板
- 尝试对接非官方API爬取

但当前阶段不做这些YAGNI特性。

## 验收标准

- [ ] `docs/nowcoder_practice.md` 创建并填入已做的题目
- [ ] 设计文档完成并通过评审
- [ ] 可选：简单统计脚本输出刷题进度

**完成后用户可以:**
- 持续添加新做的题目
- 方便复习整理知识点
- 清晰追踪刷题进展
```
