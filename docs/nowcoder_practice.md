### 1. TCP序号确认号计算 - [计算机网络] - 2026-03-29

**题目描述：**

若主机甲与主机乙之间已建立一个TCP连接，双方持续有数据传输，且数据无差错与丢失。甲收到一个来自乙的TCP段，该段的序号为800、确认序号为1000、有效载荷为50字节，则甲立即发送给乙的TCP段的序号和确认序号分别是（）

**选项：**
- A. 1000、849
- B. 1000、850
- C. 1001、850
- D. 1001、851

**我的思考：**


**正确答案：** B

**知识点总结：**

TCP序号和确认号的基本规则：
1. **序号(sequence number)**：本报文段**第一个字节**的序号。对方发来的确认号告诉我们对方期望收到我们的下一个字节是什么，所以我们发出去的段序号就等于对方发来的确认号。
2. **确认号(acknowledgment number)**：期望收到对方下一个报文段**第一个字节**的序号。计算方式：确认号 = 对方发来报文段的第一个字节序号 + 对方有效载荷字节数。
3. 例子：本题中，乙的段从序号800开始，共50字节 → 覆盖序号范围是 [800, 800+50-1] = [800, 849] → 甲期望下一个从850开始，所以确认号是850。
4. 乙给甲的确认号是1000 → 乙已经收到了甲的前999字节，期望甲接下来发序号1000开始的数据 → 所以甲发出去的段序号是1000。

**结果：** 序号 = 1000，确认号 = 850。

### 2. Linux命令删除空行 - [Linux/Shell] - 2026-03-30

**题目描述：**

Linux 中有 `time.txt` 文件，内容如下：
```
1
2
3
4
5
6
7
8
2021
2020

1993
2010


2030
```
想要将文件中的内容去除所有空行输出，则下面命令正确的是（）

**选项：**
- A. `cat -s time.txt | grep -v ' '`
- B. `cat -n time.txt | grep -v ' '`
- C. `cat time.txt | grep -v '^$'`
- D. `cat time.txt | grep -v '^@'`

**我的思考：**


**正确答案：** C

**知识点总结：**

`grep` 正则表达式匹配规则：
- `^` 匹配行首，`$` 匹配行尾
- `^$` 正好匹配"从行首到行尾之间什么都没有"，也就是**空行**
- `-v` 参数表示"反选"，输出不匹配的行 → `grep -v '^$'` 就是输出所有**不是**空行的行

其他选项错在哪里：
- A/B: `grep -v ' '` 是去掉**包含空格**的行，而不是空行。而且原题中 `cat` 命令换行错误，语法不对
- D: `^@` 匹配以@开头的行，和空行无关

补充：`cat -s` 的作用是**将连续多个空行合并成一个空行**，并不能删除所有空行。
