## 📅 2026-04-01 (今日)

### 1. LeetCode 283. 移动零
- **状态**：✅ 已掌握核心逻辑
- **知识点**：双指针法、原地修改数组、空间复杂度O(1)优化。
- **解法对比**：
  - 覆盖法：遍历数组将非0元素移到前面，最后补0，直观易懂
  - 交换法：双指针直接交换0和非0元素，一次遍历完成，更高效
- **关键理解**：慢指针记录非0元素的放置位置，快指针遍历数组，不需要额外数组空间。
- **收获**：双指针是数组原地操作的核心模式，适用于移动、去重等场景。

### 2. LeetCode 66. 加一
- **状态**：✅ 已掌握核心逻辑
- **知识点**：进位处理、数组逆序遍历、大数处理技巧。
- **解法对比**：
  - 转换法：数组转数字加一再转数组，简单但不适用于超大数据范围
  - 进位法：逆序遍历处理进位，原地修改，空间复杂度O(1)，最优解法
- **关键理解**：遇到9才需要进位，全9的情况需要在数组开头加1。
- **收获**：处理进位问题时，逆序遍历是常用技巧，避免重复修改前面的元素。

### 3. LeetCode 1822. 数组元素积的符号
- **状态**：✅ 已掌握核心逻辑
- **知识点**：符号判断、无实际乘积计算、边界条件处理。
- **解法**：遍历数组，遇到0直接返回0，遇到负数翻转符号，最终返回符号值。
- **关键理解**：不需要计算实际乘积，只需统计负数个数和检查是否存在0，避免大数溢出。
- **收获**：很多问题可以通过数学性质简化计算，避免不必要的复杂度。

### 1. LeetCode 242. 有效的字母异位词
- **状态**：✅ 已掌握核心逻辑
- **知识点**：哈希计数、数组模拟哈希表、Unicode 字符处理。
- **解法对比**：
  - 排序法：O(n log n) 时间，简单直观适合快速验证
    ```python
    def isAnagram(self, s: str, t: str) -> bool:
        return sorted(s) == sorted(t)
    ```
  - 计数法：O(n) 时间，最优解法，利用数组下标映射26个字母
- **进阶思考**：处理Unicode字符时用字典代替固定数组，保持O(n)时间复杂度
- **收获**：哈希计数是处理频率类问题的通用模式，也是大数据统计的基础。

### 2. LeetCode 389. 找不同
- **状态**：✅ 已掌握核心逻辑
- **知识点**：异或运算、ASCII码总和法、字符计数。
- **解法对比**：
  - 异或法：O(n) 时间，O(1) 空间，最巧妙
    ```python
    def findTheDifference(s: str, t: str) -> str:
        res = 0
        for c in s + t:
            res ^= ord(c)
        return chr(res)
    ```
  - 总和法：O(n) 时间，O(1) 空间，直观易懂
- **关键理解**：这两种方法只适用于"恰好多出1个字符"的场景，不能用在异位词判断等需要精确计数的场景，否则会有哈希碰撞。
- **收获**：掌握了异或运算的适用边界，理解了"有损压缩"和"无损统计"的区别。

### 3. LeetCode 459. 重复的子字符串
- **状态**：✅ 已掌握核心逻辑
- **知识点**：字符串拼接技巧、KMP前缀函数、周期性字符串判断。
- **解法对比**：
  - 拼接法：O(n) 时间，1行代码实现，非常巧妙
    ```python
    def repeatedSubstringPattern(self, s: str) -> bool:
        return s in (s + s)[1:-1]
    ```
  - KMP前缀函数法：O(n) 时间，最优空间复杂度的解法，需要掌握前缀函数实现
- **KMP理解要点**：
  - `j`代表当前已经匹配成功的字符个数
  - 匹配失败时`j = pi[j-1]`是复用已经匹配子串的公共前后缀，避免重复比较
  - 判断规则：`n % (n - pi[-1]) == 0` 说明字符串有周期性
- **收获**：字符串周期性问题可以通过拼接法快速验证，KMP的前缀函数是处理字符串匹配类问题的核心工具。
